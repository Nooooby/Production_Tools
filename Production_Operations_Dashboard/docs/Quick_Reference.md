## 🔧 常见问题解决方案

### Q1: 如何修改 SKU 信息？

**当前问题**: SKU 在多个表格中维护，容易不一致

**解决方案**:
```
步骤 1: 只在 99_SKU_MASTER 中修改
步骤 2: 按 Ctrl+Shift+F9 刷新所有公式
步骤 3: 检查 05_Today's Orders 中相关行是否已更新
步骤 4: 记录在变更日志中
```

### Q2: 如何添加新订单？

**当前问题**: 需要在多个表格中输入，容易遗漏

**临时方案** (改进前):
```
步骤 1: 在 05_Today's Orders 中添加
步骤 2: 手动计算原材料需求，填入 06_ReqFrame
步骤 3: 更新 02_Tray Pack Order
步骤 4: 手动更新生产计划表 Plan
```

**建议方案** (改进后):
```
步骤 1: 在 "02_Order_Input_Form" 填写
步骤 2: 系统自动汇总和分配
步骤 3: 自动更新所有相关表格
步骤 4: 完成！
```

### Q3: 公式出错了怎么办？

**快速排查步骤**:
```
1. 按 Ctrl+` 显示公式视图
2. 在公式栏中看到公式本身
3. 检查引用的表格名称是否正确
4. 检查引用的单元格范围是否正确
5. 按 F2 逐步修改和验证

常见错误:
❌ 表名拼写错误: 'TabName' vs 'Tab Name'
❌ 范围越界: 引用 A1:Z1000 但数据只有 100 行
❌ 循环引用: A1 引用 B1，B1 引用 A1
```

### Q4: 如何生成周报告？

**步骤**:
```
1. 打开 Plan 工作表
2. 检查周数和时间范围
3. 从 06_Dashboard 复制相关数据
4. 粘贴到 Report 中的周汇总区
5. 导出为 PDF
```

**优化后** (改进版本):
```
1. 点击 "生成周报告" 按钮
2. 选择日期范围
3. 自动生成！
```

## 🆘 故障排除

### 问题: 公式显示 #REF! 错误

**原因**: 引用的单元格或工作表已删除或改名

**解决**:
```
1. 找到显示 #REF! 的单元格
2. 按 F2 或双击编辑
3. 查找出错的引用
4. 改正表名或单元格地址
5. 按 Enter 确认
```

### 问题: 文件打开很慢

**原因**: 公式过多或数据过大

**解决**:
```
临时方案:
- 关闭自动计算: 公式 → 计算选项 → 手动
- 只打开需要的表

长期方案:
- 执行 Phase 2 优化
```

### 问题: 数据不一致

**排查步骤**:
```
1. 确认 99_SKU_MASTER 是最新的
2. 按 F9 重新计算所有公式
3. 按 Ctrl+Shift+F9 强制刷新
4. 关闭不保存，重新打开
5. 如果仍有问题，使用备份恢复
```
